## 1.2.0

- Add `inpaint_image` tool: FLUX.1-Fill-dev inpainting with separate source-image and mask uploads, optional LoRA, and descriptive-prompt guidance baked into the tool description (Fill-dev is not instruction-tuned). Mask is read from the red channel so any greyscale PNG works without alpha-channel gymnastics.
- Add `read_image_metadata` tool: extracts the text prompt, model, seed, sampler, steps, and LoRA from PNGs generated by ComfyUI or Automatic1111. Handles FLUX pipelines that route through `BasicGuider` → `InpaintModelConditioning` → `CLIPTextEncode`.
- Add `count` parameter to `generate_image` for batch generation (each image gets `base_seed + index`, output filenames get a numeric suffix).
- Flip the default `return_image` on both `generate_image` and `inpaint_image` from `"none"` to `"thumb"` so the calling model sees its output without having to ask.
- `submit_prompt` now surfaces ComfyUI's actual rejection body (`node_errors` / `error`) instead of a bare "No prompt_id in response", making workflow-authoring bugs much easier to diagnose.
- Docs: FLUX inpainting research notes (variant comparison, ControlNet primer, Fill-dev prompting convention) and a Fill-dev Q8_0 download section in the ComfyUI setup guide.
